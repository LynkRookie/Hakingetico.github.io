Fase 3: Acceso

En esta etapa, el atacante intenta explotar vulnerabilidades específicas para obtener acceso al sistema o red. Aquí es donde el atacante "rompe" la seguridad y compromete un recurso

¿Cómo funciona?

Explotación directa de vulnerabilidades: El atacante utiliza errores o configuraciones inseguras en software, sistemas operativos o aplicaciones para acceder al sistema.

Exploración de red: Determina la topología de la red, detecta dispositivos conectados y verifica si están Manipulación de datos: Se aprovecha de entradas inseguras, como formularios web, para inyectar código o manipular las operaciones del sistema. Desbordamiento de memoria: Se envía más información de la que un programa puede manejar, causando fallos que el atacante puede controlar.

Objetivos principales: Obtener acceso inicial al sistema. Escalar privilegios para controlar funciones administrativas. Acceder a datos sensibles, recursos del sistema, o funcionalidades clave.

Impacto en la seguridad: El acceso exitoso significa que las defensas iniciales del sistema han fallado. Dependiendo del nivel de acceso obtenido, el atacante puede moverse lateralmente por la red, instalar malware o robar datos.

Tipos de Ataques

XSS (Cross-Site Scripting)

Inyección de scripts maliciosos en páginas web.

SQL Injection

Inserción de código SQL malicioso para acceder a bases de datos.

Prevención

  • Validar y sanear entradas del usuario.
  • Utilizar consultas parametrizadas para prevenir inyecciones SQL.
  • Escapar caracteres especiales en formularios para prevenir XSS.
