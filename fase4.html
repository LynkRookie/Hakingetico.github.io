Fase 4: Mantenimiento de Acceso

Después de obtener acceso, el atacante busca formas de garantizar su permanencia en el sistema sin ser detectado, permitiendo la ejecución de futuras acciones maliciosas.

¿Cómo funciona?

Persistencia: El atacante establece mecanismos para reingresar al sistema aun si la vulnerabilidad inicial es parcheada o el sistema se reinicia. Esto puede incluir modificar configuraciones o instalar programas ocultos.

Evasión: El atacante oculta su presencia para evitar ser detectado por herramientas de seguridad o monitoreo.

Control remoto: Se asegura de poder interactuar con el sistema desde ubicaciones externas.

Objetivos principales:: Garantizar el acceso continuo al sistema. Minimizar el riesgo de detección por parte de administradores o herramientas de seguridad. Mantener la capacidad de realizar acciones adicionales, como espionaje, robo de datos o sabotaje.

Impacto en la seguridad: El mantenimiento de acceso implica un riesgo continuo, ya que el sistema comprometido puede ser utilizado para ataques futuros, espionaje prolongado o para comprometer otras partes de la red.
